Step 1: Check Status

Check which files have changed.

git status


Step 2: Add Files (git add)
Adds changes to the Staging Area (prepares files for commit).
Add one file:   git add index.html
Add all files:  git add .

Staging Area: A temporary area where changes are prepared before committing.

Step 3: Commit Changes (git commit)
Saves the staged changes as a snapshot in the local repository.
git commit -m "Added login page"

Always write a short, meaningful commit message.
Examples:
"Fixed navbar bug"
"Updated README"
"Added contact form"


Step 4: Push Changes (git push)
Uploads your local commits to the remote GitHub repository.

git push origin main
origin → Remote repository name (default)
main → Branch name


Complete Workflow
git status
git add .
git commit -m "Your commit message"
git push origin main
