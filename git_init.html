Git Notes

From git init to git push origin main

1. What is Git?

Git is a Distributed Version Control System (DVCS) that helps developers track changes in source code and collaborate with others.

Advantages of Git

  • Track code changes
  • Store project history
  • Work with teams
  • Restore previous versions
  • Manage branches

2. Initialize a Git Repository

Command

git init

Purpose

Creates a new Git repository in the current project folder.

Example

mkdir MyProject
cd MyProject
git init

Output

Initialized empty Git repository

3. Check Repository Status

Command

git status

Purpose

  • Shows untracked files
  • Shows modified files
  • Shows staged files
  • Shows files ready for commit

Example Output

Untracked files:
index.html

4. Create Project Files

Example

touch index.html

or create files manually.

Check Status

git status
Untracked files:
index.html

5. Add Files to Staging Area

Add One File

git add index.html

Add Multiple Files

git add index.html style.css script.js

Add Everything

git add .

Purpose

Moves files from the Working Directory to the Staging Area.

Check Status

git status
Changes to be committed:
new file: index.html

6. Commit Changes

Command

git commit -m "Initial commit"

Purpose

Saves a permanent snapshot of all staged files.

Example

git commit -m "Created homepage"

7. View Commit History

Complete History

git log

Short Version

git log --oneline

Example

a25f9f1 Initial commit

8. Configure Git (First Time Only)

Set Username

git config --global user.name "Your Name"

Set Email

git config --global user.email "you@example.com"

View Configuration

git config --list

9. Create Repository on GitHub

  1. Login to GitHub
  2. Click New Repository
  3. Enter Repository Name
  4. Click Create Repository
  5. Copy Repository URL

Example URL

https://github.com/username/myproject.git

10. Connect Local Repository to GitHub

Command

git remote add origin https://github.com/username/myproject.git

Check Remote

git remote -v

Example Output

origin  https://github.com/username/myproject.git (fetch)
origin  https://github.com/username/myproject.git (push)

11. Rename Branch to Main

Some older Git versions create the default branch as master. Rename it to main. 

git branch -M main

Check Current Branch

git branch
* main

12. Push Project to GitHub

First Push

git push -u origin main

Meaning

  • push → Upload commits
  • -u → Set upstream branch
  • origin → Remote repository
  • main → Branch name

Next Time

git push

Complete Git Workflow

git init

git status

git add .

git commit -m "Initial commit"

git branch -M main

git remote add origin https://github.com/username/repository.git

git push -u origin main

Daily Git Workflow

git status

git add .

git commit -m "Describe your changes"

git push

Git Workflow Diagram

Working Directory │ │ git add ▼ Staging Area │ │ git commit ▼ Local Repository │ │ git push ▼ GitHub Repository

Common Git Commands

Command Description
git init Initialize Git Repository
git status Check repository status
git add . Stage all files
git add filename Stage specific file
git commit -m "message" Create a commit
git log Show commit history
git log --oneline Short commit history
git branch Show branches
git branch -M main Rename branch to main
git remote add origin URL Connect local repository with GitHub
git remote -v View remote repository
git push -u origin main Push project first time
git push Push latest commits
git pull Download latest changes
git clone URL Download an existing repository

Interview Questions

1. What is Git?

Git is a Distributed Version Control System used to track changes in source code.

2. What does git init do?

Creates a new Git repository in the current directory.

3. What is the Staging Area?

A temporary area where files are prepared before committing.

4. Difference between git add and git commit?

  • git add stages changes.
  • git commit permanently saves staged changes.

5. What is origin?

The default name of the remote GitHub repository.

6. What does git push do?

Uploads local commits to GitHub.

7. What does git branch -M main do?

Renames the current branch to main.

8. Why use git status?

To check the current state of files and staging area.

Practice Exercise

  1. Create a folder named GitPractice.
  2. Run git init.
  3. Create an index.html file.
  4. Run git status.
  5. Stage the file using git add ..
  6. Commit using git commit -m "Initial commit".
  7. Create a GitHub repository.
  8. Connect it using git remote add origin <repository-url>.
  9. Rename the branch using git branch -M main.
  10. Push the project using git push -u origin main.
Tip: Follow this sequence every day while working on a project: git status → git add → git commit → git push.
