Git & GitHub Notes

Basics to Clone Repository & Check Repository Status

1. What is Git?

Git is a Distributed Version Control System (DVCS). It helps developers keep track of changes made in project files over time. Git stores every version of your project so that you can restore previous versions whenever required.

Why do we use Git?

  • Track changes in source code.
  • Restore previous versions of files.
  • Work safely without losing code.
  • Allow multiple developers to work together.
  • Create separate branches for new features.
  • Merge completed work into the main project.
Example:
Suppose you accidentally delete some code today. Git allows you to restore yesterday's version easily.

2. What is GitHub?

GitHub is a cloud-based platform that hosts Git repositories online. It allows developers to store projects, collaborate with teams, and back up their code.

Remember:
  • Git → Software used to track code.
  • GitHub → Online website where Git repositories are stored.
Git GitHub
Version Control Tool Cloud Hosting Platform
Works Offline Requires Internet
Installed on Computer Website for Collaboration

3. Repository (Repo)

A Repository is a project folder that Git tracks. It contains:

  • Project files
  • Folders
  • Git history
  • Commits
  • Configuration files

Types of Repository

  1. Local Repository
  2. Remote Repository

4. Local Repository

A Local Repository is stored on your own computer. You can edit files and use Git commands even without an internet connection.

Example Folder

MyProject/

Advantages

  • Works offline
  • Fast operations
  • Safe local development

5. Remote Repository

A Remote Repository is stored on GitHub. It is mainly used for:

  • Backup
  • Team collaboration
  • Sharing projects
  • Accessing projects from multiple devices

Example

https://github.com/username/project-name

Git Workflow

Working Directory │ ▼ Local Git Repository │ ▼ Remote GitHub Repository
Working Directory → Where you write code.
Local Repository → Git stores project history on your computer.
Remote Repository → GitHub stores your project online.

6. Install Git

Download Git from the official website: 

https://git-scm.com/

Check Installation

git --version

Example Output

git version 2.45.0

7. Configure Git (First Time Only)

Before using Git, configure your username and email. These details appear in every commit you make.

Set Username

git config --global user.name "Shivam"

Set Email

git config --global user.email "example@gmail.com"

View Configuration

git config --list

8. Clone Repository

What is Clone?

Clone means making a complete copy of a GitHub repository on your local computer.

Syntax

git clone <repository-url>

Example

git clone https://github.com/shivam0-1/learn_Git_Github_Apna_College.git

What Happens After Cloning?

  • Downloads all project files.
  • Downloads complete Git history.
  • Creates a Local Repository.
  • Automatically connects to the Remote Repository.

9. Git Status

The git status command displays the current state of your repository. It is one of the most frequently used Git commands.

Command

git status

It Shows

  • Current branch name.
  • Modified files.
  • New (untracked) files.
  • Deleted files.
  • Files ready to commit (staged).
  • Files not yet staged (unstaged).

Example (Clean Repository)

On branch main

nothing to commit,
working tree clean

Example (After Editing a File)

Changes not staged for commit:

modified: index.html
Always run git status before making commits. It helps you know exactly what has changed.
